How to Use the Prompt Evaluation System

Understanding the Evaluation Metrics

Accuracy (45% of total score)

Measures how well the AI's response matches the requirements:

  • Semantic similarity between expected and actual outputs
  • Completeness of the response
  • Relevance to the prompt

Complexity (35% of total score)

Evaluates the technical sophistication of the response:

  • Technical depth (OOP concepts, async operations, algorithms)
  • Component complexity (functions, control structures, error handling)
  • Integration complexity (dependencies, API endpoints, database operations)

Reliability (20% of total score)

Assesses the robustness and quality of the code:

  • Error handling and security measures
  • Test coverage and quality
  • Edge case handling

How to Evaluate Your Prompts

  1. Write your prompt in the Google Doc editor below
  2. Use the "===" separator to divide different parts of your prompt:
    • Before "===": Write your main prompt or requirements
    • After "===": Write your expected output or example
  3. The system will automatically evaluate the prompt based on:
    • Clarity and specificity of requirements
    • Technical depth and complexity
    • Expected reliability and robustness
  4. View your evaluation results in the metrics bar above

Tips for Better Prompts

  • Be specific about requirements and constraints
  • Include examples of expected outputs after the "===" separator
  • Specify technical requirements and frameworks
  • Mention any edge cases to consider
  • Define success criteria clearly
